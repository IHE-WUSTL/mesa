Introduction

The MESA tests include a number of cases each of which rely on a sequence of messages between actors. These tests range across different integration profiles:

  • Scheduled Patient Workflow
  • Patient Information Reconciliation
  • Presentation of Grouped Procedures
  • Charge Posting
  • Postprocessing Workflow
  • Reporting Workflow

This document should be used in parallel with the document IHE Tests: Transaction Sequences. This document describes the attributes in PPS (Modality PPS and General Purpose PPS) messages and values expected by the MESA tests.

GPPS Message from Report Manager to DSS/OF or Image Manager

This is the status update message used by the Report Manager to inform either the DSS or Image Manager of the status of the interpretation step. The Technical Framework says this status update message correponds to an unscheduled GPPS and is not a copy of the GPPS message described in the table above. The MESA tests assume that many values can be copied by the Report Manager. The table below lists only those attributes that are different.
The first difference is in the SOP Instance UID of this GPPS. Because this is not a copy of the GPPS produced by the Report Creator, the Report Manager generates a new SOP Instance UID for this GPPS.
