Patient Context Participant Tests

Introduction

The MESA tests include a number of cases each of which rely on a sequence of messages between actors. These tests range across different integration profiles:

  • Retrieve Information for Display
  • Enterprise User Authentication
  • Patient Identity Cross Reference for MPI
  • Patient Synchronized Application
  • Consistent Time

This document lists the transactions and messages for a number of cases. It may not describe the clinical scenario behind each case, but listing the transactions should clearly define what is expected of each actor.  These are all of the transactions involving all of the actors.  When you test with your particular actor, you may see only a subset of these messages.

Patient Identification

There are four test patients used in these tests. The patients have patient identifiers that are created by two different assigning authorities: MIR and WUSTL. The Patient Context Participant under test knows about identifiers created by the MIR assigning authority. The participant may see identifiers from both the MIR and WUSTL authorities, but should operate on the MIR identifiers.
Because the IHE PSA profile uses the Patient.Id.IdList context data, the Patient Context Participant may find the identifier for the patient in either Patient.Id.IdList.1 or Patient.Id.IdList.2. It is up to the participant to examine both context items to find the appropriate identifier.

Configuration

  1. Find the context data file in the Sentillion installation directory. Using the default installation, this should be C:\Program Files\ Sentillion\Sdk\WindowsComponents.
  2. Copy the Sentillion file vdkData.mdb so you can refer back to their test data and test cases.
  3. Replace the Sentillion file vdkData.mdb with the MESA file $MESA_TARGET/runtime/mdb/IHEPSA_vdk.mdb. That is, copy IHEPSA_vdk.mdb and overwrite the Sentillin vdkData.mdb file.

Starting the MESA Servers

The MESA servers are actually the Sentillion servers installed with their SDK. Once installed, these are started automatically.

Test Cases: PSA

This section describes test cases that are generally associated with the PSA Integration Profile. There may be some overlap with other profiles.

Test Case 10701: Patient Context Test 1

In test case 10701, the Patient.Id.IdList.1 value uses the patient identifier assigning authority known to the Patient Context Participant. That is the MIR value. The application under test starts with no patient context. The context simulator chooses the patient Ward, Maria. The application under test should accept that context and select that patient.

References

Instructions

  1. Start the application under test. It should have no patient context selected.
  2. Start two copies of the Sentillion Simulator.
  3. Use one copy of the Sentillion Simulator to select the patient Ward, Maria. That should propose a context with Patient.Id.IdList.1 equal to 2345^^^MIR.
  4. The second Sentillion Simulator should change context to Ward, Maria.
  5. The application under test should change context to Ward, Maria.

Evaluation

To evaluate your response to this test:

  1. Obtain a screen capture (digital camera, screen capture) showing both Sentillion simulator applications and the application under test showing the patient Ward, Maria.
  2. Name the screen capture 10701.xxx, where xxx is the appropriate file extension.
  3. Submit the picture (email) to the Technical Project Manager.

Test Case 10703: Patient Context Test 3

In test case 10703, one patient is selected two different times. The first time, Patient.Id.IdList.1 uses the MIR identifier. On the second selection, the Patient.Id.Idlist.2 context contains the MIR identifier.

References

Instructions

  1. This test starts right after the completion of test 10702. The context of both Sentillion simulators and the appliation under test should refer to Taylor, Edward.
  2. Use one copy of the Sentillion Simulator to select the patient King, Brian (first instance in the list). That should propose a context with Patient.Id.IdList.1 equal to 6521^^^MIR.
  3. The second Sentillion Simulator should change context to King, Brian.
  4. The application under test should change context to King, Brian.
  5. Use the Sentillion Simulator to select the patient Ward, Maria. The application under test should swith to that patient.
  6. Use the Sentillion Simulator to select the second instanced of King, Brian. The application under test should switch to King, Brian.
