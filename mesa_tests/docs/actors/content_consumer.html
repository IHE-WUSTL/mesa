Content Consumer

Introduction

The MESA tests include a number of cases each of which rely on a sequence of messages between actors. These tests range across different integration profiles:

  • XDS-MS
  • XDS-SD
  • PPHP
  • EDR
  • BPPC
  • XPHR
  • XD*-Lab

This document lists the transactions and messages for a number of cases. It may not describe the clinical scenario behind each case, but listing the transactions should clearly define what is expected of each actor.  These are all of the transactions involving all of the actors.  When you test with your particular actor, you may see only a subset of these messages.

Note that the tests for Content profiles such as XDS-MS and XDS-SD are over and above tests for the delivery of data for profiles XDS, XDM and XDR. The tests described in this document concern the content described in the XDS-MS and XDS-SD profile. The XDS/XDM/XDR tests concerned with delivery of data are documented elsewhere.

Test Cases: XDS-SD

This section describes test cases that are generally associated with the XDS-SD Integration Profile.

Test Case 14121: XDS-SD: Render Sample Documents

In test 14121, the Document Consumer renders a PDF/A and a plaintext document and submits the two screen shots to the Project Manager for evaluation. The purpose of this test is to make sure the consumer application can properly render the XDS-SD document.

References

Instructions

  1. The file $MESA_TARGET/mesa_tests/iti/msgs/xds-sd/example1.xml represents a sample CDA-wrapped PDF/A document according to the XDS-SD Integration Profile.

    $MESA_TARGET/mesa_tests/iti/msgs/xds-sd/example2.xml     is CDA-wrapped plaintext.

    Import these 2 documents into your system by out of band means (the MESA tests do not give you a Document Repository that allows you to retrieve this document).

  2. Render the content on your system.

  3. Perform a screen capture of the rendered output. Name the captured output <your-system-name>_14121_pdf.<extension> and <your-system-name>_14121_text.<extension> where <extension> is html, jpg, bmp, doc, etc.

Evaluation

  1. Submit the file created above to the Project Manager. The Project Manager will examine to determine sufficient information has been made available to the user.

Test Case 14101: Sample XDS-SD Documents

The Document Consumer will import a number of documents supplied by different XDS-SD Document Sources. The Document Consumer fills out a worksheet indicating if the content could be imported/rendered.
The purpose of this test is to give Document Consumers a chance to examine content in advance of a face to face test. This will allow some debugging of document content to be performed offline rather than in real time.

References

Instructions

  1. Look on the MESA Software Distribution web page for the file: xds-sd-14101.zip. This file contains CDA documents from Document Source actors and a Word document that is a worksheet to complete.
  2. Import/render each CDA-wrapped document in the zip file.
  3. For each document, fill out the appropriate data in the worksheet.
  4. Name the worksheet: <your-system-name>_14101.xls

Evaluation

  1. Submit the worksheet created above to the Project Manager for evaluation.
  2. If you find any documents that are problematic, contact the Project Manager as soon as possible so we can work with the source of the data.

Test Cases: PCC

This section describes test cases that are generally associated with several different PCC profiles.

40800: PCC XPHR Render Aortic Dissection

In test 40800, the Content Consumer is asked to "process/render" a test document from the Aortic Dissection test case. You need to use your imagination for this test. Process/render means to apply one or more of the options that are listed below:

  • View
  • Document Import
  • Section Import
  • Discrete Data Import

You will perform one or more of those actions on the sample document and then provide evidence that you have performed this action. That evidence will be some screen capture or database dump from your system.

References

Instructions

  1. The file $MESA_TARGET/mesa_tests/pcc/msgs/aortic/ad-xphr.xml represents a sample XPHR document according to the XPHR Integration Profile.
  2. Import this document into your system by out of band means.
  3. Render/process the content on your system. Generate some display (GUI, database dump, command line output) that demonstrates that you have successfully processed the document.
  4. Perform a screen capture of the rendered output. Name the captured output <your-system-name>_40800<extension> and where <extension> is html, jpg, bmp, doc, etc. If the output is just a text file, then you can just submit the text file without performing a screen capture.
  5. Create a separate text file (.txt, not word document) that briefly explains which option is supported and how that is demonstrated by the output in the step above.
  6. Place both files (output file, text explanation) in a zip file (name_40800.zip) and submit that zip file for evaluation.

Evaluation

  1. Submit the zip file created above to the Project Manager. The Project Manager will examine to determine sufficient information has been made available to the user.
