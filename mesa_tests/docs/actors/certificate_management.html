Introduction

The MESA tests for the Basic Security integration profile require the use of digital certificates to identify/authenticate actors. This document describes how these certificates are managed for the MESA tests. This does not include how the certificates are managed for Connectathons, exhibits, demonstrations or in clinical settings.

MESA and “System Under Test” Certificates

The MESA actors will use a single certificate to identify all of the MESA actors. This is not the normal case. One would expect the different MESA actors to use different digital certificates. This is done in the interest of simplicity.
We make the same simplifying assumption for the system under test. The MESA tools include one certificate to be used for all systems under test.

Installing Digital Certificates

The MESA software (binary) is shipped with the digital certificates installed in $MESA_TARGET/certificates. Your application will need a key value and digital certificates. These are stored in the files: test_sys_1.key.pem and test_sys_1.cert.pem. There is a second key/certificate pair named test_sys_2. You can use those files for testing your system. The MESA system as shipped is configured to accept either certificate from systems under test.
The MESA software will present one of two certificates to your application. These are found in the same directory. The certificates are found in the files mesa_1_cert.pem and mesa_2_cert.pem. You should load these into your system so that you recognize these certificates from the MESA applications.
Some organizations would prefer to generate their own certificates or obtain certificates from a third party. If you wish to test this, you need to configure the MESA software to recognize that certificate. Place a copy of that certificate in $MESA_TARGET/runtime/certificates/test_list.cert. That should be sufficient (although you probably will have to restart MESA servers). If that does not work, email a copy of your certificate to the Project Manager and ask for assistance.

Generating New Certificates

This section is intended for Project Managers who distribute certificates. End users are not expected to generate their own certificates (but are welcome to if they so desire).
The scripts for generating certificates are found in $MESA_ROOT/tls. The script cert_v01.csh generates the certificates used for MESA actors, the system under test and some special certificates, such as an expired certificate. Run the script cert_v01.csh.
