Introduction

The MESA tests include a number of cases each of which rely on a sequence of messages between actors. These tests range across different integration profiles:

  • Retrieve Information for Display
  • Enterprise User Authentication
  • Patient Identity Cross Reference for MPI
  • Patient Synchronized Application
  • Consistent Time

This document lists the transactions and messages for a number of cases. It may not describe the clinical scenario behind each case, but listing the transactions should clearly define what is expected of each actor.  These are all of the transactions involving all of the actors.  When you test with your particular actor, you may see only a subset of these messages.

Test Case 10301: EUA Kerberos Authentication

Test case 10301 covers the EUA Kerberos Authentication. A user “logs-in” to a client computer using Kerberos authentication with a Kerberos Authentication Server.

References

ITI TF-2: 3.2

Instructions

This test does not involve any MESA software. The intent of this test is for your computer system to support a user login with name/password authentication managed by a Kerberos Authentication Server.

UNIX Workstation Instructions

  1. Create a Kerberos Authentication Server. We have a separate document (Kerberos Notes) with notes about how to do this. We imagine most people will create this server using Windows 2000 Server, but that is not a requirement.
  2. Create an account entry on the authentication server with the name “khimss”. Set the password to something of your choosing.
  3. Configure your client system to use Kerberos authentication with the Kerberose Authentication Server. Your client computer should not use local password verification (see our document Kerberos Notes.
  4. Logout from your client system. Unplug the network adapter of that system so it cannot reach the Kerberos Authentication Server. Try to login to your computer using the khimss account and password. If this is successful, it indicates you are not authenticating with the Kerberos Authentication Server; check your configuration and try again.
  5. Reconnect your client system to your local network. Login to your computer using the khimss account and password you selected.
  6. From a terminal emulator, run the command klist. This should print ticket information for your account. Redirect the output to the text file grade_10301.txt:
       klist  > grade_10301.txt

Email the grade file to the Project Manager.

Windows Workstation Instructions

  1. Create a Kerberos Authentication Server. We have a separate document (Kerberos Notes) with notes about how to do this. We imagine most people will create this server using Windows 2000 Server. On the Authentication Server, create a domain called HIMSS.
  2. Create an account entry on the authentication server with the name “khimss”. Set the password to something of your choosing.
  3. Configure your client system to use Kerberos authentication with the Kerberose Authentication Server. (Domain name is HIMSS).Your client computer should not use local password verification (see our document Kerberos Notes.
  4. Logout from your client system. Unplug the network adapter of that system so it cannot reach the Kerberos Authentication Server. Try to login to your computer using the khimss account and password. If this is successful, it indicates you are not authenticating with the Kerberos Authentication Server; check your configuration and try again.
  5. Reconnect your client system to your local network. The login screen should allow you to choose a domain. Based on step 1 above, the domain HIMSS should be available. Use a camera (preferably digital) to take a photograph of the login screen showing the HIMSS domain. Submit/email the photograph to the Project Manager.
  6.  Login to your computer using the khimss account and password you selected.

Evaluation

The goal of this test is to make sure you know how to configure your computer system to use Kerberos authentication and that you have tested that in your lab. No further testing of Kerberized communication is required.
