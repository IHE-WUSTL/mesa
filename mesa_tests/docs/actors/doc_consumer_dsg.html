Document Consumer

Introduction

The MESA tests include a number of cases each of which rely on a sequence of messages between actors. These tests range across different integration profiles:

  • XDS-MS
  • XDS-SD

This document lists the transactions and messages for a number of cases. It may not describe the clinical scenario behind each case, but listing the transactions should clearly define what is expected of each actor.  These are all of the transactions involving all of the actors.  When you test with your particular actor, you may see only a subset of these messages.

Note that the tests for Content profiles such as XDS-MS and XDS-SD are over and above tests for the delivery of data for profiles XDS, XDM and XDR. The tests described in this document concern the content described in the XDS-MS and XDS-SD profile. The XDS/XDM/XDR tests concerned with delivery of data are documented elsewhere.

Test Cases: XDS-MS

This section describes test cases that are generally associated with the XDS-MS Integration Profile. There may be some overlap with other profiles.

Test Case 12521: XDS-MS: Render Sample Document

Not avalailable in 10.3.0 release.

In test 12521, the Document Consumer renders a sample document and submits the screen shot to the Project Manager for evaluation. The purpose of this test is to make sure the consumer renders enough information as required by the XDS-MS Integration Profile.

References

Instructions

  1. The file $MESA_TARGET/mesa_tests/iti/msgs/xds-ms/example1.xml represents a sample CDA document according to the XDS-MS Integration Profile. Import this document into your system by out of band means (the MESA tests do not give you a Document Repository that allows you to retrieve this document).
    1. Render with existing style sheet (required) – style sheet can be downloaded from $MESA_TARGET/mesa_tests/iti/actors/doc_cons/12521
    2. Render on your own (optional)
  2. Render the content on your system.
  3. Perform a screen capture of the rendered output. Name the captured output <your-system-name>_12521.<extension> where <extension> is html, jpg, bmp, doc, etc.

Evaluation

  1. Submit the file created above to the Project Manager. The Project Manager will examine to determine sufficient information has been made available to the user.

Test Case 12522: XDS-MS: Content Quiz on One Document

Not avalailable in 10.3.0 release.

The Document Consumer is given a sample CDA document and is asked to fill out a brief questionnaire concerning the content of the document.

References

Instructions

  1. The file $MESA_TARGET/mesa_tests/iti/msgs/xds-ms/example1.xml represents a sample CDA document according to the XDS-MS Integration Profile. Import this document into your system by out of band means (the MESA tests do not give you a Document Repository that allows you to retireve this document).
  2. Render the content on your system.
  3. Locate the file $MESA_TARGET/mesa_tests/iti/actors/doc_cons/12522/grade_12522.txt. This file contains questions concerning the content of the document. Modify the file by answering the questions.

Evaluation

  1. Submit the modified 12522/grade_12522.txt to the Project Manager for evaluation.

Supplemental Information

In many MESA tests, the software indicates you have 0 errors. In this test, you will have to determine the proper answers to questions and wait for later evaluation. All answers should be in the XML document.

Test Case 12523: Render Vendor Documents

Not avalailable in 10.3.0 release.

The Document Consumer will import a number of documents supplied by different Document Sources. The Document Consumer fills out a worksheet indicating if the content could be imported/rendered.
The purpose of this test is to give Document Consumers a chance to examine content in advance of a face to face test. This will allow some debugging of document content to be performed offline rather than in real time.

References

Instructions

  1. Look on the MESA Software Distribution web page for the file: xds-ms-12523.zip. This file contains CDA documents from Document Source actors and a Word document that is a worksheet to complete.
  2. Import/render each CDA document in the zip file.
  3. For each document, fill out the appropriate data in the worksheet.
  4. Name the worksheet: <your-system-name>_12523.doc

Evaluation

  1. Submit the worksheet created above to the Project Manager for evaluation.
  2. If you find any documents that are problematic, contact the Project Manager as soon as possible so we can work with the source of the data.
